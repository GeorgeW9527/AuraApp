# ☁️ Aura 云端功能使用指南

## 📖 概述

Aura 现在支持完整的云端功能，包括用户认证和数据同步。所有数据都安全地存储在 Firebase 云端，可以在多个设备间自动同步。

## ✨ 功能亮点

### 1. 用户认证
- **邮箱注册**：使用邮箱和密码创建账号
- **安全登录**：密码加密存储，支持密码重置
- **自动登录**：记住登录状态，无需重复登录
- **退出登录**：随时退出账号

### 2. 数据同步
- **营养记录同步**：拍摄的食物照片和分析结果自动保存到云端
- **跨设备访问**：在任何设备上登录，都能看到完整的历史记录
- **实时更新**：数据变化立即同步到所有设备
- **安全存储**：所有数据加密传输和存储

### 3. 云端存储
- **照片存储**：食物照片自动上传到 Firebase Storage
- **数据备份**：所有健康数据安全备份在云端
- **永久保存**：数据不会因为卸载应用而丢失

## 🚀 快速开始

### 第一次使用

1. **启动应用**
   - 打开 Aura 应用
   - 看到登录界面

2. **注册账号**
   - 点击"立即注册"
   - 输入邮箱地址（例如：`your@email.com`）
   - 设置密码（至少 6 位）
   - 可选：输入昵称
   - 点击"注册"按钮

3. **开始使用**
   - 注册成功后自动登录
   - 进入主界面，开始使用所有功能

### 登录已有账号

1. 打开应用
2. 输入邮箱和密码
3. 点击"登录"
4. 自动恢复所有云端数据

### 忘记密码

1. 在登录界面输入邮箱
2. 点击"忘记密码？"
3. 查收邮件中的重置链接
4. 设置新密码

## 💡 使用技巧

### 1. 保持登录状态
- 应用会记住你的登录状态
- 除非主动退出，否则无需重复登录
- 即使关闭应用，下次打开仍然是登录状态

### 2. 网络要求
- 首次登录需要网络连接
- 数据同步需要网络连接
- 离线时可以查看已缓存的数据（未来功能）

### 3. 多设备使用
- 可以在多个设备上登录同一账号
- 所有设备的数据自动同步
- 建议在主要设备上使用

### 4. 数据管理
- 定期查看和清理不需要的记录
- 保持云端存储空间充足
- 重要数据建议定期截图备份

## 🐛 常见问题

### Q: 注册失败怎么办？
A: 检查以下几点：
- 邮箱格式是否正确
- 密码是否至少 6 位
- 网络连接是否正常
- 该邮箱是否已被注册

### Q: 登录失败怎么办？
A: 可能的原因：
- 邮箱或密码输入错误
- 账号不存在（需要先注册）
- 网络连接问题
- 尝试使用"忘记密码"重置

### Q: 数据没有同步？
A: 检查：
- 是否已登录账号
- 网络连接是否正常
- 查看 Xcode 控制台的日志
- 尝试重新登录

### Q: 照片上传失败？
A: 可能原因：
- 照片太大（应用会自动压缩）
- 网络不稳定
- Firebase Storage 配置问题
- 查看错误提示信息

### Q: 如何删除账号？
A: 当前版本暂不支持，未来版本会添加此功能。
如需删除数据，可以：
1. 在应用中删除所有记录
2. 联系开发者手动删除

## 📊 数据使用情况

### Firebase 免费额度
- **Firestore**：
  - 存储：1 GB
  - 读取：50,000 次/天
  - 写入：20,000 次/天
  
- **Storage**：
  - 存储：5 GB
  - 下载：1 GB/天

### 预估使用量
- 每张照片：约 100-300 KB（压缩后）
- 每条营养记录：约 1 KB
- 1000 条记录 + 照片：约 300 MB

对于个人使用，免费额度完全够用！

## 🔧 开发者信息

### 技术栈
- **认证**：Firebase Authentication
- **数据库**：Cloud Firestore
- **存储**：Firebase Storage
- **前端**：SwiftUI + MVVM

### 数据结构
```
Firestore:
  nutritionRecords/
    {userId}/
      items/
        {recordId}/
          - foodName
          - calories
          - protein, carbs, fat
          - imageURL
          - timestamp

  userProfiles/
    {userId}/
      - displayName
      - email
      - health data
```

### API 使用
所有云端操作都通过 `FirebaseManager` 单例管理：
- `signUp()` - 注册
- `signIn()` - 登录
- `signOut()` - 退出
- `saveData()` - 保存数据
- `fetchData()` - 获取数据
- `uploadImage()` - 上传图片
